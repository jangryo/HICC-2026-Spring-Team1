# 2026 Spring 1주차
## 1.웹 서버
* HTML, 이미지, 데이터 등 거의 모든 형태의 데이터는 **HTTP**로 주고 받는다. 서버간에도
* **웹 서버** - 정적 리소스(이미지와 같은 이미 만들어진 것들) 클라이언트와 주고받는데 사용
* **웹 애플리케이션 서버(WAS)** - 웹 서버 + 코드를 싪행하는 동적인 행동까지 가능
* WAS만으로 구성시 과부하, 다운 시 에러화면도 노출 불가능
  - 따라서 클라이언트 - 웹 서버 - WAS - DB 식으로 구성해 분업 및 죽어도 웹 서버에서 오류 화면을 띄울 수 있게 함

## 2.서블릿
* HTTP 프로토콜은 TCP/IP 커넥션, 소켓 연결 등을 포함해 모든 업무에 잡일이 많음. 이를 우리가 전부 쓰기는 귀찮음
  - 따라서 의미있는 코드만 써도 이런 잡일들을 대신해주는게 서블릿 (using namespace std 같은 일 줄여주는 거)
### 서블릿 컨테이너의 동작 원리
1. **사용자 요청**: WAS는 `Request`, `Response` 객체를 새로 생성함.
2. **서블릿 실행**: 설정된 서블릿을 호출하며 생성된 객체들을 전달.
3. **로직 수행**: 서블릿이 `Request`에서 데이터를 꺼내 쓰고, 결과를 `Response`에 담음.
4. **자원 정리**: 응답이 완료되면 사용된 `Request`, `Response` 객체는 소멸함.

### 서블릿(Servlet) vs 서블릿 컨테이너(Container)
- **서블릿**: 개발자가 작성한 '비즈니스 로직' 클래스 (요리사)
- **서블릿 컨테이너**: 서블릿의 생명주기를 관리하고 요청을 전달하는 '실행 환경' (주방 / 예: Tomcat) + 멀티 쓰레드 지원
  
* 요청에 따라 일시적인 request, response 객체 생성 vs 싱글톤(서버 생성시 하나만 생성->돌려씀)인 서블릿객체
  - 때문에 공유변수 사용에 주의해야한다.
 
## 3. 멀티 쓰레드
* **쓰레드**란 커넥션 연결 후 서블릿 객체를 호출해주는 친구
* 즉 코드를 하나씩 커리하는 애, 얘 없으면 안돌아감
* 요청 하나면 상관 없으나 요청은 많이 들어는데 처리지연이 되거나 하면 죽어버림

***쓰레드를 요청마다 만들면은?***
 - 비용이 너무 많이 들어 리소스가 많이 든다.
 - 컨텍스트 스위칭 비용( CPU가 쓰레드를 옮겨다닐 때 생기는 비용)이 무지 비쌈

**쓰레드 풀**
- 이를 해결하기 위해 미리 서버에 부하가 가지 않는 선에서 많은 쓰레드를 만들어 쓰레드 풀에 저장함
- 요청 시 쓰레드 풀에서 대여, 사용이 끝나면 반납하는 구조
 ---------------------
* 쓰레드 생성, 제거에 필요한 자원이 없어지고 최대치가 정해져 많은 요청이 들어와도 죽지 않는다 (쓰레드 수에 비해 과한 요청은 드랍)
* WAS가 멀티 쓰레드를 지원해 개발자는 싱글 쓰레드마냥 편하게 개발하되 **객체의 멤버 변수 공유가 안되게만** 주의 할 것

***따라서 이 쓰레드 수를 적절히 관리하는게 주요 튜닝 포인트***

## 4. HTML, HTTP API, CSR, SSR
### 우리가 고려해야하는 것
**1. 정적 리소스**
- 별 다른 가공 없이 고정된 파일을 제공하는 것이고 누가 언제 요청해도 같은게 보임
**2. HTML 페이지**
  - 서버가 실시간으로 HTML을 생성해서 보내는 방식
**3. HTTP API**
    - HTML이 아닌 데이터(json)을 넘겨줌
    - ui는 어플, 웹 프레임 워크 등 다른 플랫폼에서 따로 만들고 우리는 데이터만 넘기면 될 때

### SSR- 서버 사이드 렌더링
 - 주로 정적인 화면에 사용, 서버가 다해서 HTML을 많들고 웹에서 그냥 보여줌.
 - 관련기술은 요즘은 타임리프->백앤드
### CSR- 클라이언트 사이드 렌더링
 - 주로 동적인 화면에서 사용, JS를 사용해 웹에서 부분변경 가능 ex) 구글지도
 - React같은 프론트 엔트

***CSR의 과정***
1. HTML을 요청 받으면 서버는 대신 JS 링크를 보냄
2. 클라이언트가 JS 요청하면 서버는 JS 로직과 HTML 랜더링 코드를 보냄
3. HTTP API 데이터를 요청하면 서버는 JSON 데이터를 보냄
4. JS 로 웹에서 결과를 랜더링
